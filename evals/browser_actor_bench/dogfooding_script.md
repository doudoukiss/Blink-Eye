# Dogfooding script

## Session 1 — Long planning

1. Start `./scripts/run-local-browser-melo.sh`.
2. Open `/client/`.
3. Speak a 60–90 second project-planning request.
4. Confirm the UI shows listening, transcript/hints, thinking, speech chunks, and memory/persona effects.

## Session 2 — Camera grounding

1. Put an object in camera view.
2. Ask what Blink sees.
3. Confirm camera status changes to looking and that the answer traces frame seq/age.
4. Ask a non-visual question and confirm camera is not falsely used.

## Session 3 — Interruption and repair

1. Ask for a long answer.
2. Interrupt with “wait / 等等”.
3. Correct a misunderstanding.
4. Confirm floor state, interruption status, stale chunk drops, and repair behavior.

Fill `human_rating_form.md` after each session.
